# Product

## Register

product

## Users

Logos developers and operators inspecting Logos testnet and local-node state while debugging node, sequencer, indexer, account, program, block, transaction, wallet, channel, messaging, and storage behavior. They are in task-focused debugging or verification sessions and need dense, reliable data with clear navigation between linked chain entities.

## Product Purpose

Logos Inspector is a native inspection toolkit for Logos networks. It helps users inspect Logos Blockchain and Logos Execution Zone state first, with a design that can extend to messaging and storage. Success means users can quickly locate current network health, open blocks and transactions, inspect accounts and decoded IDL data, follow references, and understand local/testnet endpoint status without switching tools.

## Brand Personality

Sharp.

## Anti-references

This should not feel like SaaS marketing, a decorative blockchain explorer clone, a toy dashboard, or a generic dark admin template. Avoid ornamental effects, marketing hero patterns, over-carded layouts, vague copy, and decorative motion that does not clarify state.

## Design Principles

- Make linked chain data navigable.
- Prefer dense, stable inspection surfaces over decorative layouts.
- Surface endpoint and data-source status at the point of use.
- Keep decoding program-agnostic and explicit about missing IDL or source limitations.
- Align with Logos Basecamp native UI conventions while preserving inspector-grade data density.

## Accessibility & Inclusion

Target WCAG AA contrast. Keep keyboard operation viable for core controls, preserve visible focus states, avoid color-only status communication where possible, and keep status palettes legible for common color-vision deficiencies. Motion should be limited to useful state feedback and respect reduced-motion expectations.
